2.1 Complexity of Algorithms

When solving a problem, there may be multiple algorithmic solutions. To choose the best algorithm for a particular task, we need to analyze and compare their efficiency. This is where complexity analysis becomes essential.

Step Count Method

The Step Count Method counts the exact number of operations needed for an algorithm as a function of the problem size (n).

Random Access Machine (RAM) Model

The step count method is based on the RAM model with the following assumptions:

  • Single processor machine: One operation at a time
  • No shared memory: Each operation is independent
  • Sequential execution: Instructions executed one after another
  • No concurrent operations: No parallel processing

What Counts as a Step?

Each atomic operation is counted as one step:

  • Arithmetic operations (+, -, *, /)
  • Comparison operations (==, <, >, !=)
  • Assignment operations (=)
  • Return statements
  • Memory access

Example 1: Simple Counting

int count(int n) { sum = 0; // 1 step for (i = 1; i <= n; i++) { // n steps (initialization) // n+1 comparisons // n increments sum += n + 1 - i; // 3n steps (subtract, add, assign) } return sum; // 1 step }
Total Steps = 1 + n + (n+1) + n + 3n + 1 = 6n + 3

Example 2: Optimized Version

int count_3(int n) { sum = n * (n + 1) / 2; // 4 steps return sum; // 1 step }
Total Steps = 5

Observation: The optimized version is significantly faster! (5 steps vs 6n+3 steps)

Example 3: Nested Loops

int count_1(int n) { sum = 0; // 1 step for (i = 1; i <= n; i++) { for (j = i; j <= n; j++) { sum++; // executed Œ£(n-i+1) times } } return sum; // 1 step }
Total Steps = (n¬≤ + 3n + 2) / 2

üí° Key Insight

While exact step counts are useful, they are:

  • Too dependent on programming languages
  • Too dependent on programmer's style
  • Often unnecessary for practical comparisons

What matters more: How the algorithm scales with input size!

2.2 Asymptotic Notation

Asymptotics is the study of functions of a parameter n, as n becomes larger and larger without bound. It focuses on the behavior of functions for very large inputs.

Why Asymptotic Analysis?

Instead of an exact step count, we want notation that:

  • Accurately reflects the increase of computation time with size
  • Ignores details that have little effect on the total
  • Provides a simple, meaningful comparison

2.6 Space Complexity vs Time Complexity

Time Complexity

Concerns determining the number of steps needed as a function of problem size.

Space Complexity

Concerns determining the amount of memory needed as a function of problem size.

Example: Factorial Calculation

Iterative Version:
int factorial(int n) { int result = 1; for (int i = 1; i <= n; i++) result *= i; return result; }

Time: O(n), Space: O(1)

Recursive Version:
int factorial(int n) { if (n <= 1) return 1; return n * factorial(n - 1); }

Time: O(n), Space: O(n) - due to call stack

2.7 Practical Examples

Example 1: Array Insertion

Example 2: Binary Search

üìå Chapter Summary

Key Concepts

  1. Step Count Method: Count exact operations, but often too detailed
  2. Asymptotic Analysis: Focus on growth rate as input size increases
  3. Big O (O): Upper bound - worst case scenario
    • Most commonly used in practice
    • Guarantees algorithm won't perform worse than this
  4. Big Omega (Œ©): Lower bound - best case scenario
    • Less commonly used
    • Shows minimum performance
  5. Big Theta (Œò): Tight bound - exact growth rate
    • Most precise notation
    • Use when algorithm performs consistently
  6. Rules for Analysis:
    • Drop constants: O(5n) = O(n)
    • Drop lower terms: O(n¬≤ + n) = O(n¬≤)
    • Nested loops multiply complexities
    • Sequential steps add complexities
  7. Common Complexities (from best to worst):
    • O(1) < O(log n) < O(n) < O(n log n) < O(n¬≤) < O(n¬≥) < O(2^n) < O(n!)
  8. Space vs Time: Often there's a trade-off between memory usage and execution speed

üí° Practical Advice

  • Always analyze worst-case complexity first
  • O(n log n) is often the best achievable for comparison-based sorting
  • Avoid O(2^n) algorithms for large inputs
  • O(n¬≤) is acceptable for small inputs (n < 1000)
  • Cache and preprocessing can improve practical performance
