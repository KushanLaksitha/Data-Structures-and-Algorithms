5.1 Array-Based Lists

Introduction to Array-Based Lists

An array-based list (or dynamic array) is an implementation of the list ADT using arrays. It combines the benefits of arrays with dynamic sizing capabilities.

Key Features:

  • Sequential storage in contiguous memory
  • Dynamic resizing when capacity is reached
  • Fast random access O(1)
  • Variable number of elements
Array-Based List Implementation
/* Array-Based List Implementation */
#include <stdio.h>
#include <stdlib.h>

#define INITIAL_CAPACITY 10

typedef struct {
    int *data;
    int size;
    int capacity;
} ArrayList;

/* Initialize the list */
void initList(ArrayList *list) {
    list->data = (int*)malloc(INITIAL_CAPACITY * sizeof(int));
    list->size = 0;
    list->capacity = INITIAL_CAPACITY;
}

/* Resize the array when needed */
void resize(ArrayList *list) {
    list->capacity *= 2;
    list->data = (int*)realloc(list->data, list->capacity * sizeof(int));
}

/* Insert at end */
void append(ArrayList *list, int value) {
    if(list->size == list->capacity) {
        resize(list);
    }
    list->data[list->size++] = value;
}

5.4 Doubly Linked Lists

Introduction to Doubly Linked Lists

A doubly linked list has nodes with two pointers: one to the next node and one to the previous node. This allows traversal in both directions.

Advantages:

  • Bidirectional traversal
  • Easier deletion (no need to track previous node)
  • Can traverse backward from any node

Disadvantages:

  • Extra memory for previous pointer
  • More complex insertion/deletion logic

Doubly Linked List Implementation
/* Doubly Linked List */
#include <stdio.h>
#include <stdlib.h>

/* Node structure with prev and next pointers */
typedef struct Node {
    int data;
    struct Node *prev;
    struct Node *next;
} Node;

/* Create a new node */
Node* createNode(int data) {
    Node *newNode = (Node*)malloc(sizeof(Node));
    newNode->data = data;
    newNode->prev = NULL;
    newNode->next = NULL;
    return newNode;
}

/* Insert at beginning */
Node* insertAtBeginning(Node *head, int data) {
    Node *newNode = createNode(data);
    
    if(head != NULL) {
        head->prev = newNode;
        newNode->next = head;
    }
    
    return newNode;
}

/* Insert at end */
Node* insertAtEnd(Node *head, int data) {
    Node *newNode = createNode(data);
    
    if(head == NULL) {
        return newNode;
    }
    
    Node *current = head;
    while(current->next != NULL) {
        current = current->next;
    }
    
    current->next = newNode;
    newNode->prev = current;
    
    return head;
}

/* Delete a node */
Node* deleteNode(Node *head, Node *delNode) {
    if(head == NULL || delNode == NULL) {
        return head;
    }
    
    /* If node to be deleted is head */
    if(head == delNode) {
        head = delNode->next;
    }
    
    /* Change next only if not last node */
    if(delNode->next != NULL) {
        delNode->next->prev = delNode->prev;
    }
    
    /* Change prev only if not first node */
    if(delNode->prev != NULL) {
        delNode->prev->next = delNode->next;
    }
    
    free(delNode);
    return head;
}

/* Display forward */
void displayForward(Node *head) {
    Node *current = head;
    printf("Forward: ");
    while(current != NULL) {
        printf("%d ", current->data);
        current = current->next;
    }
    printf("\n");
}

/* Display backward */
void displayBackward(Node *head) {
    Node *current = head;
    
    if(head == NULL) {
        return;
    }
    
    /* Go to last node */
    while(current->next != NULL) {
        current = current->next;
    }
    
    printf("Backward: ");
    while(current != NULL) {
        printf("%d ", current->data);
        current = current->prev;
    }
    printf("\n");
}

int main() {
    Node *head = NULL;
    
    head = insertAtEnd(head, 10);
    head = insertAtEnd(head, 20);
    head = insertAtEnd(head, 30);
    head = insertAtBeginning(head, 5);
    
    displayForward(head);
    displayBackward(head);
    
    return 0;
}
5.5 Circular Linked Lists

Introduction to Circular Linked Lists

A circular linked list is a variation where the last node points back to the first node, forming a circle. There is no NULL at the end.

Applications:

  • Round-robin scheduling
  • Multiplayer games (turn-based)
  • Music playlist (repeat mode)
  • Browser history navigation

Circular Linked List Implementation
/* Circular Linked List */
#include <stdio.h>
#include <stdlib.h>

typedef struct Node {
    int data;
    struct Node *next;
} Node;

Node* createNode(int data) {
    Node *newNode = (Node*)malloc(sizeof(Node));
    newNode->data = data;
    newNode->next = NULL;
    return newNode;
}

/* Insert at beginning */
Node* insertAtBeginning(Node *head, int data) {
    Node *newNode = createNode(data);
    
    if(head == NULL) {
        newNode->next = newNode;  /* Point to itself */
        return newNode;
    }
    
    /* Find last node */
    Node *current = head;
    while(current->next != head) {
        current = current->next;
    }
    
    newNode->next = head;
    current->next = newNode;
    
    return newNode;
}

/* Insert at end */
Node* insertAtEnd(Node *head, int data) {
    Node *newNode = createNode(data);
    
    if(head == NULL) {
        newNode->next = newNode;
        return newNode;
    }
    
    /* Find last node */
    Node *current = head;
    while(current->next != head) {
        current = current->next;
    }
    
    current->next = newNode;
    newNode->next = head;
    
    return head;
}

/* Delete a node by value */
Node* deleteNode(Node *head, int key) {
    if(head == NULL) {
        return NULL;
    }
    
    Node *current = head;
    Node *prev = NULL;
    
    /* If head needs to be deleted */
    if(head->data == key) {
        /* Find last node */
        while(current->next != head) {
            current = current->next;
        }
        
        if(head == head->next) {
            free(head);
            return NULL;
        }
        
        current->next = head->next;
        free(head);
        return current->next;
    }
    
    /* Search for node */
    prev = head;
    current = head->next;
    while(current != head && current->data != key) {
        prev = current;
        current = current->next;
    }
    
    if(current->data == key) {
        prev->next = current->next;
        free(current);
    }
    
    return head;
}

/* Display circular list */
void displayList(Node *head) {
    if(head == NULL) {
        printf("List is empty\n");
        return;
    }
    
    Node *current = head;
    printf("Circular List: ");
    
    do {
        printf("%d ", current->data);
        current = current->next;
    } while(current != head);
    
    printf("\n");
}

int main() {
    Node *head = NULL;
    
    head = insertAtEnd(head, 10);
    head = insertAtEnd(head, 20);
    head = insertAtEnd(head, 30);
    head = insertAtBeginning(head, 5);
    
    displayList(head);
    
    head = deleteNode(head, 20);
    printf("After deleting 20:\n");
    displayList(head);
    
    return 0;
}
Comparison of List Types

Time Complexity Comparison

Operation Array-Based Singly Linked Doubly Linked Circular Linked
Access by Index O(1) O(n) O(n) O(n)
Insert at Beginning O(n) O(1) O(1) O(n)*
Insert at End O(1) O(n) O(n) O(n)
Delete at Beginning O(n) O(1) O(1) O(n)*
Delete at End O(1) O(n) O(1)** O(n)
Search O(n) O(n) O(n) O(n)

* O(n) due to finding last node to update circular link
** O(1) if tail pointer is maintained

Space Complexity Comparison

List Type Space per Element Notes
Array-Based List Data only Most space efficient, but may waste allocated space
Singly Linked List Data + 1 pointer Extra pointer overhead
Doubly Linked List Data + 2 pointers Highest memory overhead
Circular Linked List Data + 1 pointer Same as singly linked

When to Use Each Type

Array-Based List

  • Need fast random access by index
  • Size is relatively stable
  • Memory efficiency is important
  • Cache performance matters

Singly Linked List

  • Frequent insertions/deletions at beginning
  • Size changes frequently
  • Don't need random access
  • Memory is fragmented

Doubly Linked List

  • Need bidirectional traversal
  • Implementing undo/redo functionality
  • Browser history navigation
  • LRU cache implementation

Circular Linked List

  • Round-robin scheduling
  • Multiplayer game turns
  • Circular buffers
  • Music/video playlists
Summary

Key Takeaways

  • Array-Based Lists: Fast access, fixed/growable size, contiguous memory
  • Singly Linked Lists: Dynamic size, efficient insertion at head, one-way traversal
  • Sorted Linked Lists: Maintain order automatically, O(n) insertion
  • Doubly Linked Lists: Bidirectional traversal, easier deletion, more memory
  • Circular Linked Lists: No end node, useful for cyclic operations
  • Trade-offs: Memory vs flexibility vs access speed
  • Choose based on: Access patterns, insertion/deletion frequency, memory constraints
Important: Understanding different list implementations helps you choose the right data structure for your specific use case. Consider the operations you'll perform most frequently when making your choice!
