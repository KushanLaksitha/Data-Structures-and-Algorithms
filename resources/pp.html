Big O notation is a mathematical notation used to describe the upper bound of an algorithm's time or space complexity in terms of input size (n). It represents the worst-case scenario of algorithm performance.

Significance:

  1. Performance Prediction: Allows comparison of algorithms independent of hardware or implementation details
  2. Scalability Analysis: Helps determine how algorithms perform as data size grows
  3. Algorithm Selection: Enables developers to choose appropriate algorithms for specific problem sizes
  4. Optimization Guide: Identifies performance bottlenecks in code

b) Time Complexities of Array-Based List Operations (4 marks)

Insertion:

  • At end (with space): O(1)
  • At beginning/middle: O(n) - requires shifting elements
  • Example: Inserting at index 0 requires moving all n elements

Deletion:

  • At end: O(1)
  • At beginning/middle: O(n) - requires shifting elements
  • Example: Deleting first element requires shifting remaining n-1 elements

Search:

  • Unsorted array: O(n) - linear search required
  • Sorted array: O(log n) - binary search possible
  • Direct access by index: O(1)

c) Static vs Dynamic Data Structures (4 marks)

Static Data Structures:

  • Fixed size determined at compile time
  • Memory allocated on stack or fixed heap allocation
  • Cannot grow or shrink during execution
  • Examples: Arrays, fixed-size structures
  • Advantages: Faster access, predictable memory usage
  • Disadvantages: Memory waste, inflexibility

Dynamic Data Structures:

  • Size can change during runtime
  • Memory allocated dynamically on heap
  • Can grow and shrink as needed
  • Examples: Linked lists, dynamic arrays (vectors), trees
  • Advantages: Flexible memory usage, efficient for unknown data sizes
  • Disadvantages: Slower access, memory management overhead

a) Array vs Linked List (4 marks)

Array:

  • Contiguous memory allocation
  • Fixed or pre-allocated size
  • Direct access: O(1) using index
  • Memory: Single block allocation

Linked List:

  • Non-contiguous memory allocation
  • Dynamic size
  • Sequential access: O(n) to reach nth element
  • Memory: Scattered nodes with pointers

Key Differences:

  1. Memory: Arrays use continuous memory; linked lists use scattered memory with pointer overhead
  2. Access Time: Arrays O(1) for any element; linked lists O(n) for nth element
  3. Insertion/Deletion: Arrays O(n) due to shifting; linked lists O(1) if position known
  4. Memory Flexibility: Arrays fixed; linked lists grow/shrink dynamically

b) Search Time Complexity Comparison (4 marks)

Unsorted Array:

  • Must check each element sequentially
  • Best case: O(1) - element at first position
  • Average case: O(n/2) = O(n)
  • Worst case: O(n) - element at end or not found

Singly Linked List:

  • Must traverse from head node
  • Best case: O(1) - element at head
  • Average case: O(n/2) = O(n)
  • Worst case: O(n) - element at end or not found
Conclusion: Both have the same time complexity O(n) for unsorted data. However, arrays may have slight performance advantage due to:
  • Better cache locality
  • No pointer dereferencing overhead
  • Sequential memory access patterns

c) Remove Element from Singly Linked List (4 marks)

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
};

// Function to remove element at given position
void removeAtPosition(Node** head, int position) {
    // Check if list is empty
    if (*head == NULL) {
        cout << "List is empty" << endl;
        return;
    }
    
    Node* temp = *head;
    
    // If removing first node
    if (position == 0) {
        *head = temp->next;
        delete temp;
        return;
    }
    
    // Traverse to node before position
    for (int i = 0; temp != NULL && i < position - 1; i++) {
        temp = temp->next;
    }
    
    // If position is out of range
    if (temp == NULL || temp->next == NULL) {
        cout << "Position out of range" << endl;
        return;
    }
    
    // Remove node at position
    Node* nodeToDelete = temp->next;
    temp->next = nodeToDelete->next;
    delete nodeToDelete;
}

d) Doubly Linked List vs Singly Linked List (4 marks)

Advantages of Doubly Linked List:

  1. Bidirectional Traversal: Can traverse forward and backward
  2. Faster Deletion: Can delete node in O(1) if node pointer is known (no need to find previous)
  3. Easier Implementation: Some operations like reverse traversal are straightforward
  4. Better for Certain Applications: Ideal for browser history, undo/redo operations

Disadvantages of Doubly Linked List:

  1. Extra Memory: Requires additional pointer per node (prev pointer)
  2. More Complex Operations: More pointer updates during insertion/deletion
  3. Increased Overhead: More memory allocation and management
  4. Slower Performance: Extra pointer manipulation may slow operations
For Frequent Add/Remove:
  • Use doubly linked list when: Need bidirectional access, deletion of known nodes
  • Use singly linked list when: Memory is critical, only forward traversal needed

e) Making End Insertion More Efficient (4 marks)

Solution: Maintain a Tail Pointer

Traditional Linked List: Insertion at end: O(n) - must traverse entire list

Modified Linked List with Tail Pointer:

struct LinkedList {
    Node* head;
    Node* tail;  // Pointer to last node
    int size;
};

void insertAtEnd(LinkedList* list, int data) {
    Node* newNode = new Node();
    newNode->data = data;
    newNode->next = NULL;
    
    if (list->head == NULL) {
        list->head = list->tail = newNode;
    } else {
        list->tail->next = newNode;
        list->tail = newNode;
    }
    list->size++;
}
Benefits:
  • Insertion at end: O(1) instead of O(n)
  • Only need to update tail pointer
  • Small overhead: one extra pointer in list structure

a) Stack and Queue Data Structures (4 marks)

Stack:

LIFO (Last In First Out) structure - Elements added and removed from same end (top)

Fundamental Operations:

  1. Push: Add element to top - O(1)
  2. Pop: Remove element from top - O(1)
  3. Peek/Top: View top element without removing - O(1)
  4. isEmpty: Check if stack is empty - O(1)

Queue:

FIFO (First In First Out) structure - Elements added at rear, removed from front

Fundamental Operations:

  1. Enqueue: Add element to rear - O(1)
  2. Dequeue: Remove element from front - O(1)
  3. Front: View front element - O(1)
  4. isEmpty: Check if queue is empty - O(1)

b) Queue in Operating System Process Scheduling (4 marks)

Real-World Example: CPU Process Scheduling

Scenario: An operating system manages multiple processes requesting CPU time using a ready queue.

How It Works:

  1. Process Arrival: New processes enter the ready queue (enqueue)
  2. CPU Allocation: Process at front gets CPU time (dequeue)
  3. FIFO Fairness: First arrived process executes first
  4. Round-Robin: After time slice, process goes to back of queue
Example:
Initial Queue: [Process A, Process B, Process C]
CPU takes Process A (dequeue)
New Process D arrives (enqueue)
Queue becomes: [Process B, Process C, Process D]
After A's time slice, A returns to queue
Queue: [Process B, Process C, Process D, Process A]

Benefits: Fair scheduling, prevents starvation, predictable order, easy to implement

c) Stack Implementation Using Linked List (4 marks)

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* next;
};

class Stack {
private:
    Node* top;
    
public:
    Stack() {
        top = NULL;
    }
    
    // Push operation - add element to top
    void push(int value) {
        Node* newNode = new Node();
        newNode->data = value;
        newNode->next = top;
        top = newNode;
        cout << value << " pushed to stack" << endl;
    }
    
    // Pop operation - remove element from top
    int pop() {
        if (top == NULL) {
            cout << "Stack Underflow" << endl;
            return -1;
        }
        
        Node* temp = top;
        int poppedValue = temp->data;
        top = top->next;
        delete temp;
        
        return poppedValue;
    }
    
    // Check if stack is empty
    bool isEmpty() {
        return top == NULL;
    }
    
    // Peek top element
    int peek() {
        if (top == NULL) {
            cout << "Stack is empty" << endl;
            return -1;
        }
        return top->data;
    }
};

d) Priority Queue vs Standard Queue Efficiency (4 marks)

Standard Queue:

  • FIFO order
  • Enqueue: O(1)
  • Dequeue: O(1)
  • All tasks treated equally
  • Simple implementation

Priority Queue:

  • Elements have priority levels
  • Highest priority dequeued first
  • Enqueue: O(log n) with heap
  • Dequeue: O(log n) with heap
  • Efficient for priority-based systems

Efficiency Comparison for High-Priority Tasks:

Advantages of Priority Queue:

  1. Immediate Handling: Critical tasks processed immediately, not waiting in line
  2. Better Response Time: High-priority items don't wait behind low-priority ones
  3. Resource Optimization: Important tasks get CPU/resources faster
  4. Real-world Suitability: Emergency systems, network packets, task scheduling
Example:
Standard Queue: [Low, Low, High, Low] - High waits for 2 Low tasks
Priority Queue: [High, Low, Low, Low] - High processed immediately

Tradeoff: Priority queue has O(log n) operations vs O(1) for standard queue, but provides better service for critical tasks.

e) Efficient Array-Based Queue Design (4 marks)

Circular Array Queue:

class CircularQueue {
private:
    int* arr;
    int front, rear, capacity, count;
    
public:
    CircularQueue(int size) {
        capacity = size;
        arr = new int[capacity];
        front = 0;
        rear = -1;
        count = 0;
    }
    
    // Enqueue - O(1)
    void enqueue(int value) {
        if (count == capacity) {
            cout << "Queue is full" << endl;
            return;
        }
        rear = (rear + 1) % capacity;  // Circular increment
        arr[rear] = value;
        count++;
    }
    
    // Dequeue - O(1)
    int dequeue() {
        if (count == 0) {
            cout << "Queue is empty" << endl;
            return -1;
        }
        int value = arr[front];
        front = (front + 1) % capacity;  // Circular increment
        count--;
        return value;
    }
    
    bool isEmpty() { return count == 0; }
    bool isFull() { return count == capacity; }
};
Benefits:
  • Constant-time O(1) enqueue and dequeue
  • Reuses array space efficiently
  • No memory waste from traditional array queue
  • Fixed memory allocation prevents fragmentation

a) Stable vs Unstable Sorting Algorithms (4 marks)

Stable Sorting:

Maintains the relative order of equal elements as they appeared in the input.

Examples of Stable Algorithms:

  1. Merge Sort: Preserves order during merging
  2. Bubble Sort: Adjacent swaps maintain order
  3. Insertion Sort: Inserts without changing equal element positions
Example:
Input: [(4,A), (3,B), (4,C), (3,D)]
Stable Sort: [(3,B), (3,D), (4,A), (4,C)] // B before D, A before C

Unstable Sorting:

Does not guarantee preservation of relative order of equal elements.

Examples of Unstable Algorithms:

  1. Quick Sort: Partitioning may change order
  2. Heap Sort: Heap operations don't preserve order
  3. Selection Sort: Swapping can move equal elements
Example:
Input: [(4,A), (3,B), (4,C), (3,D)]
Unstable Sort: [(3,D), (3,B), (4,C), (4,A)] // Order changed
Importance: Stability matters when sorting by multiple keys (e.g., sort by name, then by grade).

b) Recursive vs Iterative Binary Search (4 marks)

Recursive Implementation:

int binarySearchRecursive(int arr[], int left, int right, int target) {
    if (left > right)
        return -1;  // Base case: not found
    
    int mid = left + (right - left) / 2;
    
    if (arr[mid] == target)
        return mid;  // Found
    else if (arr[mid] > target)
        return binarySearchRecursive(arr, left, mid - 1, target);
    else
        return binarySearchRecursive(arr, mid + 1, right, target);
}

Iterative Implementation:

int binarySearchIterative(int arr[], int n, int target) {
    int left = 0, right = n - 1;
    
    while (left <= right) {
        int mid = left + (right - left) / 2;
        
        if (arr[mid] == target)
            return mid;
        else if (arr[mid] > target)
            right = mid - 1;
        else
            left = mid + 1;
    }
    return -1;  // Not found
}

Differences:

  1. Space Complexity: Recursive O(log n) stack space; Iterative O(1)
  2. Performance: Iterative slightly faster (no function call overhead)
  3. Readability: Recursive more intuitive; Iterative more explicit
  4. Stack Overflow: Recursive risk with deep recursion; Iterative safe

c) Merge Sort Implementation (4 marks)

#include <iostream>
using namespace std;

// Merge function is assumed to be available
void merge(int arr[], int left, int mid, int right);

void mergeSort(int arr[], int left, int right) {
    if (left < right) {
        // Find middle point
        int mid = left + (right - left) / 2;
        
        // Sort first half
        mergeSort(arr, left, mid);
        
        // Sort second half
        mergeSort(arr, mid + 1, right);
        
        // Merge sorted halves
        merge(arr, left, mid, right);
    }
}

// Wrapper function
void mergeSortArray(int arr[], int n) {
    mergeSort(arr, 0, n - 1);
}

// Example usage:
int main() {
    int arr[] = {12, 11, 13, 5, 6, 7};
    int n = sizeof(arr) / sizeof(arr[0]);
    
    mergeSortArray(arr, n);
    
    cout << "Sorted array: ";
    for (int i = 0; i < n; i++)
        cout << arr[i] << " ";
    
    return 0;
}
Time Complexity: O(n log n)
Space Complexity: O(n)

    • Worst case easily avoided in practice

a) Binary Search Tree (BST) Definition and Operations (4 marks)

Definition:

A Binary Search Tree is a binary tree where:

  • Left subtree contains nodes with values less than parent
  • Right subtree contains nodes with values greater than parent
  • Both subtrees are also BSTs
  • No duplicate values

Insertion Operation:

  1. Start at root
  2. Compare value with current node
  3. If less, go left; if greater, go right
  4. Repeat until finding empty spot
  5. Insert new node there

Time Complexity: O(h) where h is height; O(log n) average, O(n) worst

Deletion Operation:

Three cases:

  1. Leaf Node: Simply remove
  2. One Child: Replace node with its child
  3. Two Children: Find inorder successor (smallest in right subtree), replace node's value, delete successor
Node* deleteNode(Node* root, int key) {
    if (root == NULL) return root;
    
    if (key < root->data)
        root->left = deleteNode(root->left, key);
    else if (key > root->data)
        root->right = deleteNode(root->right, key);
    else {
        // Node found
        if (root->left == NULL) {
            Node* temp = root->right;
            delete root;
            return temp;
        } else if (root->right == NULL) {
            Node* temp = root->left;
            delete root;
            return temp;
        }
        // Two children: get inorder successor
        Node* temp = findMin(root->right);
        root->data = temp->data;
        root->right = deleteNode(root->right, temp->data);
    }
    return root;
}

c) Lowest Common Ancestor (LCA) Implementation (4 marks)

#include <iostream>
using namespace std;

struct Node {
    int data;
    Node* left;
    Node* right;
    
    Node(int val) {
        data = val;
        left = right = NULL;
    }
};

// Function to find LCA of two nodes p and q in BST
Node* findLCA(Node* root, int p, int q) {
    // Base case: empty tree
    if (root == NULL)
        return NULL;
    
    // If both p and q are smaller than root, LCA is in left subtree
    if (p < root->data && q < root->data)
        return findLCA(root->left, p, q);
    
    // If both p and q are larger than root, LCA is in right subtree
    if (p > root->data && q > root->data)
        return findLCA(root->right, p, q);
    
    // Otherwise, root is the LCA
    // (one value on left side, other on right side, or one equals root)
    return root;
}

// Example usage
int main() {
    // Create BST
    Node* root = new Node(20);
    root->left = new Node(10);
    root->right = new Node(30);
    root->left->left = new Node(5);
    root->left->right = new Node(15);
    root->right->left = new Node(25);
    root->right->right = new Node(35);
    
    // Find LCA of 5 and 15
    Node* lca = findLCA(root, 5, 15);
    cout << "LCA of 5 and 15: " << lca->data << endl;  // Output: 10
    
    // Find LCA of 5 and 25
    lca = findLCA(root, 5, 25);
    cout << "LCA of 5 and 25: " << lca->data << endl;  // Output: 20
    
    return 0;
}
Time Complexity: O(h) where h is height
Space Complexity: O(h) for recursion stack

e) Adjacency Matrix vs Adjacency List for Graphs (4 marks)

Adjacency Matrix:

2D array where matrix[i][j] = 1 if edge exists between vertex i and j

Advantages:

  1. Fast Edge Lookup: O(1) to check if edge exists
  2. Simple Implementation: Easy to understand and code
  3. Good for Dense Graphs: When many edges exist
  4. Easy Edge Addition/Removal: O(1) operations

Disadvantages:

  1. Space Inefficient: O(V¬≤) space regardless of edges
  2. Slow Iteration: O(V¬≤) to iterate all edges
  3. Wasteful for Sparse Graphs: Many zeros stored
  4. Not Scalable: Large memory for many vertices

Adjacency List:

Array of lists where list[i] contains all neighbors of vertex i

Advantages:

  1. Space Efficient: O(V + E) space
  2. Fast Iteration: O(V + E) to iterate all edges
  3. Good for Sparse Graphs: Only stores existing edges
  4. Scalable: Handles large graphs well

Disadvantages:

  1. Slower Edge Lookup: O(degree) to check if edge exists
  2. More Complex: Requires linked list or dynamic array management
  3. Cache Performance: Pointer chasing can be slower
  4. More Memory per Edge: Pointers add overhead
Recommendation:
  • Use Matrix: Dense graphs (E ‚âà V¬≤), frequent edge queries, small graphs
  • Use List: Sparse graphs (E << V¬≤), frequent neighbor iteration, large graphs

Example:

Graph with 4 vertices, edges: (0,1), (0,2), (1,2), (2,3)

Adjacency Matrix:

0 1 2 3 0 0 1 1 0 1 0 0 1 0 2 0 0 0 1 3 0 0 0 0

Adjacency List:
