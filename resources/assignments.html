Array Operations & Complexity Analysis

Assignment 1
Due Date
Week 3
Difficulty
Easy
Points
100
Topics
Arrays, Big O

Implement various array operations and analyze their time complexity. This assignment will help you understand array data structures and practice Big O notation analysis.

Learning Objectives

  • Understand array implementation and memory layout
  • Implement basic array operations (insert, delete, search)
  • Analyze time complexity of operations
  • Compare different implementation approaches

Requirements

  • Implement an array-based list in C/C++
  • Support operations: insert, delete, search, display
  • Handle edge cases (full array, empty array, invalid index)
  • Write complexity analysis for each operation
  • Include test cases demonstrating all operations

Deliverables

  • Source code (.c or .cpp file)
  • Report with complexity analysis (PDF)
  • Test results and screenshots
  • README with compilation instructions

üí° Tips

  • Start with simple insert and display operations
  • Test boundary conditions thoroughly
  • Remember: array deletion requires shifting elements

Topics
Linked Lists

Implement a singly linked list with all standard operations. Compare the performance with array-based lists and understand the trade-offs between different data structures.

Learning Objectives

  • Master pointer manipulation in C/C++
  • Implement dynamic memory allocation
  • Understand linked list advantages and disadvantages
  • Handle memory management properly

Requirements

  • Implement singly linked list with node structure
  • Operations: insertAtBeginning, insertAtEnd, insertAtPosition, delete, search, display
  • Implement a sorted linked list variant
  • Compare with array-based list (provide analysis)
  • Proper memory management (no memory leaks)
  • Test with at least 100 elements

Deliverables

  • Source code for singly linked list
  • Source code for sorted linked list
  • Comparative analysis report (PDF)
  • Memory leak test results
  • Performance comparison charts

üí° Tips

  • Draw diagrams before coding insertion/deletion
  • Always check for NULL pointers
  • Handle edge cases: empty list, single node, last node
  • Free memory when deleting nodes
  • Use helper functions for cleaner code

  • Algorithm explanation document

BST, Trees

  • Visualize tree structure

  • Tree visualization output
  • Performance comparison report
  • Algorithm explanations
  • Comprehensive test cases

üí° Tips

  • Start with insert and traversal
  • Draw trees on paper first
  • For deletion: find inorder successor
